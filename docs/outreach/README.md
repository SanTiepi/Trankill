## Règles d'or

- **Pas de pitch.** Tu demandes, tu écoutes, tu notes.
- **Un email = un destinataire personnalisé.** Pas de mailing groupé.
- **Relance UNE fois** après 10 jours si pas de réponse, puis stop.
- **Anonymise les témoignages** si tu les utilises ailleurs.
- **Si un email génère un meeting**, prépare 3 questions maximum. Pas un slide deck.

## Ce que tu cherches à apprendre

Pour chaque contact, la même grille mentale :

1. **Qui vois-tu en vrai ?** (persona réel du terrain, pas celui que j'imagine)
2. **Qu'est-ce qui te manque concrètement ?** (pas « ce serait bien si »)
3. **Est-ce qu'un outil qui dit *"dans le doute, appelle X ou demande à Y"* t'aiderait ?**
4. **Quelle serait la raison n°1 pour laquelle ça ne marcherait PAS dans ton contexte ?**

## Après la Phase 0

Tu reviens avec ~10-20 inputs réels, et tu décides :

- **Concept validé** → Phase 1 code (Safe Pause + consent-first circle + Web Share Target Android)
- **Concept à pivoter** → on refait un brainstorm avec les nouvelles données
- **Partenariat potentiel** → tu codes ce qu'ils demandent, pas ce que tu avais imaginé
